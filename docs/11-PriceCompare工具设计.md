# PriceCompare 工具设计

`PriceCompare` 在 `ItemSearch` 执行完成后执行。它接收多个国内平台返回的商品候选，统一计算国内最终到手价，并按最终价格从小到大排序。

## 一、执行时机

主 Agent fork 多路子 Agent，分别调用 `item_search` 拿到京东 / 拼多多 / 淘宝等多平台商品候选。

所有平台商品结果汇总到主 Loop。

汇总完成后，调用 `PriceCompare`。

```text
多路 item_search 并行检索
-> 收集多平台原始商品
-> PriceCompare 统一核算优惠价并按价格排序
-> 输出给 ItemPicker 做综合条件筛选
-> 筛选后生成最终比价清单
```

## 二、国内版核心功能

国内版 `PriceCompare` 保持单一职责：**把最终价格算清楚**。

它接收全部平台商品列表，统一计算国内真实到手价，并按价格从小到大排序：

- 读取商品原价。
- 读取店铺优惠券。
- 读取平台满减。
- 读取百亿补贴等优惠。
- 计算每件商品优惠后价格。
- 读取运费。
- 判断是否包邮。
- 计算最终到手价。
- 把全部商品按最终到手价升序重排。
- 输出排好序的商品列表。

`PriceCompare` 只输出价格维度排序，不做任何综合筛选。

最终价格口径：

```text
最终到手价 = 商品优惠后价格 + 运费
包邮时运费 = 0
```

## 三、国内版不做的功能

这些能力全部拆给其他工具：

| 不做的事 | 交给谁 |
| --- | --- |
| 物流时效、快递质量、偏远地区可达性 | 后续独立物流工具 |
| 根据评分、销量、材质、自营标签过滤商品 | `ItemPicker` |
| 结合用户长期偏好做个性化加权排序 | `ItemPicker` |
| 生成购物总结、推荐话术 | 汇总工具 |

## 四、ItemSearch 到 PriceCompare 链路

```text
ItemSearch
  -> 多平台并行检索
  -> 返回候选商品

PriceCompare
  -> 接收全部候选商品
  -> 核算优惠后价格
  -> 加上运费 / 包邮规则
  -> 按最终到手价升序排序

ItemPicker
  -> 综合评分、销量、材质、自营、用户偏好等条件筛选

ShoppingSummary
  -> 生成最终比价清单
```

## 五、待确认点

优惠和运费信息是否能拿到，取决于具体平台 API 能力。

需要分别确认：

- 京东联盟官方 API 是否提供店铺券、平台券、满减、补贴价。
- 多多进宝官方 API 是否提供券后价、补贴、佣金相关字段。
- 淘宝 / 万邦聚合是否提供券后价、优惠券、活动价、满减字段。
- 各平台 API 是否提供运费、包邮、配送地区相关字段。

如果 API 暂时拿不到完整优惠和运费信息，第一版 `PriceCompare` 可以先按已有字段计算：

```text
商品优惠后价格 = API 返回的券后价 / 活动价 / 当前价
最终到手价 = 商品优惠后价格 + 已知运费
```

缺失字段不猜测。运费缺失时标记为未知，不强行当作包邮；价格缺失时保留原价或当前价，并在结果里标记价格来源。
