# 电商工具

工具决定 Agent 是什么 Agent。前三章（架构、技术栈、AGUI 协议）放哪个 Agent 项目都能用。第四章 [通用工具](04-通用工具.md) 同样是任何一个 Agent 都需要的。**从这六个工具开始，Globex 才是一个电商选品与比价 Agent。**

## 工具明细

| # | 工具名 | 归属 | 触发阶段 | 触发时机 |
|---|--------|:--:|----------|----------|
| 1 | **CategoryInsight** | 外部 | Think | 需要获取品类行业趋势、爆款商品数据 |
| 2 | **ItemSearch** | 外部 | Think | 需要检索商品资源 |
| 3 | **ItemPicker** | 内部 | Reflect | 检索返回的商品列表需要二次筛选过滤 |
| 4 | **PriceCompare** | 外部 | Reflect | 需要跨平台商品比价 |
| 5 | **ShippingCalc** | 外部 | Act | 计算关税、物流运费（通常搭配 ItemSearch 使用） |
| 6 | **ShoppingSummary** | 内部 | Reflect | 信息完整充足，可以输出采购清单 |

## 关键区分

**内部**：不调外部 API，LLM 自身能力。ItemPicker（按偏好筛商品）、ShoppingSummary（汇总出清单）在 AgentLoop 推理中消化。

**外部**：调第三方能力。CategoryInsight（行业 RAG）、ItemSearch（电商平台搜索 API）、PriceCompare（比价接口）、ShippingCalc（物流/关税接口），做成 LangChain Tool 进 `bind_tools`。

## AgentLoop 三个阶段（电商部分）

```
Think ──→ Reflect ──→ Act
  │           │          │
  │           │          └── ShippingCalc
  │           └── ItemPicker / PriceCompare / ShoppingSummary
  └── CategoryInsight / ItemSearch
```
